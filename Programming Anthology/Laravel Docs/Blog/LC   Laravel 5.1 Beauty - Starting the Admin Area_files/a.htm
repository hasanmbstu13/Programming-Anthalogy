                                          Brad Brad • 6 months ago

                                          Also found in 5.2 that $errors is not automatically available to the views without grouping the intended routes into a group that uses the web middleware.
                                          For instance, to fix the admin/login page, the routes for auth/login would look like this:

                                          // Logging in and out
                                          $router->group([
                                          'namespace' => 'Auth',
                                          'middleware' => 'web',
                                          ], function () {
                                          Route::get('/auth/login', 'AuthController@getLogin');
                                          Route::post('/auth/login', 'AuthController@postLogin');
                                          Route::get('/auth/logout', 'AuthController@getLogout');
                                          });

                                          Edit: The web middleware must be added BEFORE the auth middleware for the Admin routing group, otherwise you will be stuck in a redirect loop. Basically the web middleware group is now the middleware that controls the session so if putting it after the auth middleware group will result in the auth group not seeing the session.

                                          $router->group([
                                          'namespace' => 'Admin',
                                          'middleware' => ['web', 'auth'],
                                          ], function () {
                                          Route::resource('admin/post', 'PostController');
                                          Route::resource('admin/tag', 'TagController');
                                          Route::get('admin/upload', 'UploadController@index');
                                          });
